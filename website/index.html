RFMS signals at a glance

Inspired by classic RFM analysis, extended for on-demand pricing—aligned with the RideWise feature schema used in training and inference.

Recency

Days since the rider’s last trip—staleness is one of the strongest churn hints.

Frequency

How often they ride; separates habitual users from occasional ones.

Monetary

Spend level helps you weigh retention ROI against acquisition cost.

Surge exposure

Share of trips during surge—proxy for price sensitivity and peak usage.

Stack & architecture

Container-friendly Python services, standard ML serialization, and clear separation between inference and presentation.

Python 3.10 · runtime FastAPI · REST API scikit-learn · Random Forest joblib · model artifact Streamlit · dashboard Docker · compose-ready
1

Data & notebooks

Cleaned trip and rider tables feed EDA and RFMS notebooks; aggregated rider records (on the order of tens of thousands of rows in the modeling set) drive train/test evaluation.

2

Model artifact

A balanced Random Forest classifier is persisted for low-latency inference alongside the five numeric features expected by the API contract.

3

Services

Uvicorn serves the FastAPI app; Streamlit (or any client) calls your deployed /predict endpoint with JSON payloads.

Contract you can rely on

The prediction service accepts rider-level aggregates and responds with a churn label and probability—simple to integrate with CRMs, in-app messaging, or internal BI.

Request

Five floats/ints: rating behavior, recency, frequency, monetary, surge exposure. 

POST /predict
Content-Type: application/json

Response

Binary churn flag plus positive-class probability.

{
  "is_churning": 0 | 1,
  "probability": 0.0 – 1.0
}

Deploy RideWise as a Render web service

This project is configured to run as a single Python Web Service on Render. Use the included render.yaml from the repo root so Render installs backend dependencies, starts FastAPI, serves this webpage at /, and keeps churn prediction live at /predict.
